Initial release. A Jellyfin Server 10.11 plugin (C#, .NET 9.0) plus a static GitHub Pages catalog viewer. The plugin subscribes to library events, debounces a quiet window, and pushes a `catalog.json` snapshot to the configured GitHub repo via the Contents API; the viewer is plain HTML+JS that fetches the JSON and renders a sortable, title-searchable table.

## Plugin

- `BasePlugin<PluginConfiguration>` + `IHasWebPages`. Plugin GUID `a12a453e-31f7-46e2-9968-d92580109f24` (frozen).
- `IPluginServiceRegistrator` registers `Debouncer`, `MediaCatalogBuilder`, `GitHubPusher` as singletons, plus a named `"github"` HttpClient via `IHttpClientFactory`, plus `LibraryEventListener` as a hosted service.
- Library events `ItemAdded` / `ItemUpdated` / `ItemRemoved` are filtered to `Movie`. Handlers return immediately; build+push runs on a background `Task` spawned by the debouncer.
- `Debouncer` uses timer-reset semantics with a dirty-during-push flag. Every state mutation under a lock; never awaits inside the lock. Default quiet window 30s, configurable.
- `GitHubPusher` calls the Contents API: GET-current-SHA then PUT, retries once on 409 with refreshed SHA, hard-fails on 401/403/422. Reads PAT via `Plugin.Instance.Configuration` on every push so rotation takes effect immediately without restart.
- `IScheduledTask` "Resync JellyWatch Now" with no recurring trigger (manual only).
- Settings page (`Web/configPage.html`, embedded resource) with Owner / Repo / Branch / JsonPath / PatToken / DebounceSeconds fields, masked PAT input with Show toggle, and a Test Connection button that hits the GitHub Contents API client-side to distinguish file-missing from repo-missing from auth-failed.
- Logging via `ILoggerFactory.CreateLogger("JellyfinMediaCatalog")` (not `ILogger<T>` per known DI resolution issues across Jellyfin 10.8-10.11).

## Viewer

- `docs/index.html` + `docs/app.js` + `docs/style.css` + `docs/catalog.json`. Vanilla JS, no build step. ~110 LOC.
- Sortable column headers, title-search filter, mobile-first responsive layout.
- Served by GitHub Pages from `main` `/docs`. The plugin's PUTs to `docs/catalog.json` are commits to main, which auto-trigger Pages rebuilds; no separate deploy step.

## Verification

- `dotnet build -c Release` zero warnings, zero errors.
- Plugin loads in portable Jellyfin 10.11.9 dev instance; appears in Dashboard -> Plugins with the right GUID; settings page renders and round-trips config values via REST.
- Resync scheduled task fires; `GitHubPusher.PushAsync` succeeds end-to-end against a sacrificial test repo; first push HTTP 201 (create), second push HTTP 200 (update via GET-SHA-then-PUT-with-SHA).
- GitHub Pages viewer renders at `https://runawaydevil.github.io/jellyfin-movie-catalog/` against the stub catalog committed to `docs/catalog.json` at scaffold time; will be overwritten by the first real push from the Unraid Jellyfin.
- Auth failure paths verified by direct REST: missing config -> `InvalidOperationException: Owner not configured`; PAT without `Contents: write` -> `GitHub push got 403; PAT bad or missing scope`.
- NUnit tests for `Debouncer` (timer-reset, dirty-during-push, FlushAsync) and `GitHubPusher` (mocked HttpMessageHandler against 200/404/409/401/422) are NOT in this release. They are tracked debt for v0.1.1.
